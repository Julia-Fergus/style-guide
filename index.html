Typography

Our base typeface is 'Helvetica'. As a baseline we will use 13px Helvetica Regular and this will also serve as the 1.0em value. Ideally we want to keep a 1.4em ration between font-size and line-height. Line height calculator

Use cases
  • H1: Main title of the UI.
  • H2: Divide sections in a UI that already have an H1.
  • H3: Titles inside sections.
  • H4: Highlight plain text.
  • Body: Plain text.
  • Small: Texts inside tags, placeholders and small spaces. Also for cases in which 13px need to be compromise for some reason.
  • Small 1: For some cases in which 11px is too small.

Buttons

A button is a call to action component.

Types

Primary Neutral Link Button Destructive

As its name implies, the primary button will be used for those actions that are consider the most important.
All the actions that don't need to be highlighted are going to be represented by a neutral button.
Links are inline text that can redirect the user to a different context from the one showing in the current UI.
Everytime that a negative action is going to be done, a destructive button will be displayed to ensure that the user is sure to perform that action.

Design specs
Height: 30px
Width: depending on the content
Padding (Left & Right): 12px
Margin: 8px
Border radius: 4px

Background color: Interaction
Font color: White
Font size: 13px
Font style: bold
Background color (on hover): #005586
Shadow (on hover): 0px 0px 3px #888888
Shadow (pressed): inset 0px 1px 2px #373737
Background color (disable): #73B3D8

Neutral button
Background color: Zero
Border color: Dust
Font color: Text
Font size: 13px
Font style: regular
Background color (on hover): #B3B3B4
Shadow (on hover): 0px 0px 3px #888888
Shadow (pressed): inset 0px 1px 2px #373737
Background color (disable): White

Link button
Font color: Interaction
Font size: 13px
Font style: regular

Destructive button
Background color: Destruction
Font color: White
Font size: 13px
Font style: regular
Background color (on hover): #952600
Shadow (on hover): 0px 0px 3px #888888
Shadow (pressed): inset 0px 1px 2px #373737
Background color (disable): #E38F73

Use cases
  • Primary button: Main actions (create a job, site visit, add variation).
  • Neutral button: Rest of the actions (schedule quote, assign person/group, add purchase order).
  • Link button: To redirect the user to another context (job number, customer name, email address).
  • Destructive button: Negative actions (void or delete).

Messages

Messages are used to inform and warn the user.

Types

This is an information message.
This is a success message.
This is a warning message.
This is an error message.

Information messages will expose data that needs to be taken into account.
Success messages are going to ensure the user that he/she did a good job.
Warning messages will warn the user that something is not as expected and there may be future problems.
Error messages will inform the user that something is not right and how to proceed to solve it.

Messages should use a friendly tone, be clear and concise.

Design specs
Width: will be the width of its parent component
Height: 32px
Icon size: 20px
Icon padding: 12px (left) 15px (right)
Icons: info-circle, check-circle, exclamation-triangle and times-circle
Background colors: Interaction, Success, Warning and Destruction in its light shade

Use cases
  • Information: to help or guide the user (invoices will be sent from this address).
  • Success: when something goes right ().
  • Warning: when something it is not completely right but can be solve easily (currently connected to Quickbooks instead of Xero).
  • Error: when something is not going as expected (archive all attached jobs and cannot be undone).
